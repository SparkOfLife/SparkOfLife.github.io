hexo+GitHub Pages 创建个人博客

前言

为什么选择hexo？

Hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github上。作者是来自台湾的tommy351。作者对hexo只有一句话的介绍:

A fast, simple & powerful blog framework
一个快速、简单并且强大的博客框架。

优点：

  • 简单：平常只需要hexo new,hexo g,hexo s,hexo d四个命令
  • 支持Markdown
  • 快：配置方便也容易理解，配置好后，马上就能看到效果。
  • 文章本地保存。
  • 兼容macOS、Windows、Linux

为什么选择GitHub Pages？

GitHub Pages本用于介绍托管在GitHub的项目，不过，由于它的空间免费稳定，用来做搭建一个博客再好不过了。

优点：

  • github pages有300M免费空间，无需自己搭建服务器
  • 配置简单
  • 就算github被墙了，我可以搬到国内的gitcafe中去

大致流程

  1. 搭建 Git 环境
  2. 搭建 Node.js 环境
  3. 安装配置 Hexo
  4. 创建Github Pages
  5. 关联 Hexo 与 GitHub Pages
  6. Hexo 的常用操作
  7. 更换主题

注：本教程是在Windows 10下进行的

搭建 Git 环境

Git 是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。我们需要利用Git将本地的网页和文章等提交到 GitHub 上。

在Git官网直接下载安装Git即可。
安装结束后在桌面右键单击出现Git Bash Here即为安装成功。

进入Git Bash Here之后输入git --version则可以显示Git的版本号。

搭建 Node.js 环境

因为 Hexo 博客系统是基于 Node.js 编写的，所以我们要搭建 Node.js 环境

安装方法同安装git差不多。在Node.js官网下载安装包安装即可。

安装结束后打开”命令提示符” 输入node -v与’npm -v’即可显示node.js与npm的版本号，安装成功。

安装配置 Hexo

随意在某处单击右键，打开Git Bash，输入

npm install hexo-cli -g

安装完成后，输入hexo version即可查看hexo版本号。

在你喜欢的文件夹里新建一个hexo文件夹（比如D：\hexo），然后在文件夹内点击鼠标右键，选择Git Bash，并且输入如下命令

hexo init

hexo就会自动在目标文件夹建立它所需要的文件了。

然后继续输入(注意此时git hash运行的位置应该在你创建的hexo文件夹内)

在安装过程中出现了几个警告，但影响不大。

继续在git hash中输入

hexo g
hexo s
即可生成并运行hexo，在浏览器输入http://localhost:4000/，就能看到自己的博客了,这标志博客已经搭建成功了，只不过现在只能在本地浏览。

创建Github Pages

首先在Github官网注册一个账号。

创建一个新的repository,名字取为your_name.github.io，注意your_name一定为你的Github账号名，不能随意取（每个账号下只能建立一个）。

关联 Hexo 与 GitHub Pages

上传博客的内容时我们可以用两种方法进行：HTTPS或SSH，其中SSH不用每次都输密码，推荐。

  • 生成新的SSH keys

命令行输入下面的内容，并回车：
ssh-keygen -t rsa -C “your_email@example.com”

将“your_email@example.com”换成你刚刚注册github时的邮箱。

然后有提示：

直接按回车键就好，然后系统要你输入密码：

这个密码是在你提交项目时使用，为空的话就不用输入，为了方便我就没输密码，直接按了回车键。
最后会看到这样的界面，这就表明设置成功了：

  • 添加SSH key到Github上

用文本编辑器打开～/.ssh/id_rsa.pub，这就是刚刚生成的秘钥,然后全选里面的内容并复制。

登陆github网站：
点击右上角的Account Settings—->SSH Public keys —-> add another public keys
把刚刚复制的内容粘贴到key里，title随便填。

点击add SSH，这就完成了。

  • 测试是否添加成功

Git hash下输入：

ssh -T git@github.com

你可能看到警告（也可能没有）：

这个不用担心，直接输入yes就好，然后会看到：

Hi username! You’ve successfully authenticated, but GitHub does not provide shell access.

出现这就表明添加成功了。

  • 设置用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。
Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字和邮箱是github注册时使用的那个。

git config —global user.name “your user_name”

git config —global user.email “your_email@example.com”

Git hash下输入：

hexo g

hexo d

如果最后出现：

[info] Deploy done: github

就说明部署成功。但如果出现下图所示情况表明部署失败，请检查上面输入的内容是否正确。

用浏览器打开SparkOfLife.github.io(将SparkOfLife改成自己的)，就能看见自己的博客了，别人也可以访问了。

Hexo 的常用操作

hexo new [layout] <title>写新文章

hexo publish [layout] <filename> 发表草稿

hexo g 生成静态页面

hexo d 部署博客

hexo s 打开本地服务器，查看博客

更换主题

hexo有很多主题，大家可以在hexo.io或GitHub上自行搜索。安装过程大家可以参考主题的README.md文件。

我使用的主题是Louis Barranqueiro制作的Tranquilpeak主题，感谢Louis Barranqueiro。大家可以打开我的博客https://sparkoflife.github.io/看一下效果。
